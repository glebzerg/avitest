### BUG-001: P0 — Некорректное отображение цены в карточке товара

**Приоритет:** P0 (Критический)  
**Серьёзность:** Critical

**Описание:**  
В третьей карточке товара "Персидский котёнок девочка" цена "15 000 ₽" перекрывается тултипом "Не показывайте мне похожие объявления", что делает цену нечитаемой.

**Шаги воспроизведения:**
1. Открыть страницу поиска котят в Москве
2. Навести курсор на иконку сердечка третьей карточки
3. Обратить внимание на отображение цены

**Фактический результат:**  
Цена "15 000 ₽" перекрывается тултипом, текст нечитаем.

**Ожидаемый результат:**  
Тултип должен отображаться рядом с иконкой, не перекрывая важную информацию (цену).

**Окружение:**
- Браузер: Chrome (по скриншоту)
- Разрешение: Десктоп (1920x1080 или выше)

**Скриншот:**  
![BUG-001](bags.png) — третья карточка, перекрытие цены

### BUG-002: P0 — Несоответствие фильтра по локации

**Приоритет:** P0 (Критический)  
**Серьёзность:** Major

**Описание:**  
В выдаче по запросу "Котята в Москве" присутствует объявление с локацией "Новосибирск" (первая карточка "Персидский котёнок 20 000 ₽").

**Шаги воспроизведения:**
1. Выполнить поиск "Котята в Москве"
2. Установить фильтр по локации "Москва"
3. Проверить локацию в первой карточке

**Фактический результат:**  
В выдаче присутствует объявление из Новосибирска.

**Ожидаемый результат:**  
Все объявления в выдаче должны соответствовать выбранной локации (Москва).

**Возможные причины:**
- Некорректная работа фильтра по локации
- Ошибка в индексации объявлений
- Проблема с геолокацией продавца

### BUG-003: P1 — Некорректное значение фильтра "Возраст"

**Приоритет:** P1 (Высокий)  
**Серьёзность:** Major

**Описание:**  
В фильтре "Возраст" для категории "Кошки" выбрано значение "Щенок", что является некорректным — "щенок" относится к собакам, для кошек должно быть "Котёнок".

**Шаги воспроизведения:**
1. Открыть страницу поиска в категории "Кошки"
2. Посмотреть на фильтр "Возраст"

**Фактический результат:**  
Отображается значение "Щенок".

**Ожидаемый результат:**  
Для категории кошек должно отображаться значение "Котёнок".

**Возможное решение:**  
Динамически изменять текст фильтра в зависимости от выбранной категории животного.

### BUG-004: P1 — Некорректное отображение заголовка страницы

**Приоритет:** P1 (Высокий)  
**Серьёзность:** Minor

**Описание:**  
В заголовке "Котята в Москве 19" цифра "19" (вероятно, количество объявлений) отображается слишком близко к слову "Москве", без пробела или с неправильным форматированием.

**Фактический результат:**  
"Котята в Москве 19" — цифра прилегает к тексту.

**Ожидаемый результат:**  
"Котята в Москве (19)" или "Котята в Москве — 19 объявлений" с корректным форматированием.

---

### BUG-005: P1 — Некорректное расположение иконки избранного

**Приоритет:** P1 (Высокий)  
**Серьёзность:** Minor

**Описание:**  
Во второй карточке "Персидский котёнок девочка 40 000 ₽" иконка сердечка (добавление в избранное) расположена слишком близко к тексту, отсутствует отступ.

**Фактический результат:**  
Иконка сердечка прилегает к тексту без отступа.

**Ожидаемый результат:**  
Между текстом и иконкой должен быть отступ (padding) минимум 8px.

---

### BUG-006: P2 — Несоответствие количества объявлений в кнопке фильтра

**Приоритет:** P2 (Средний)  
**Серьёзность:** Minor

**Описание:**  
Кнопка "Показать 8 объявлений" внизу фильтров показывает неверное количество — на странице видно больше 8 объявлений.

**Фактический результат:**  
Кнопка отображает "8 объявлений", но на странице видно минимум 9 карточек.

**Ожидаемый результат:**  
Количество в кнопке должно соответствовать фактическому количеству отображаемых объявлений.

---

### BUG-007: P2 — Обрезанный текст в фильтре

**Приоритет:** P2 (Средний)  
**Серьёзность:** Trivial

**Описание:**  
В левом фильтре текст "Рейтинг 4 звезды и выше" обрезан и не полностью виден.

**Фактический результат:**  
Текст обрезан, видна только часть фразы.

**Ожидаемый результат:**  
Текст должен полностью отображаться или переноситься на новую строку.

---

### BUG-008: P2 — Отсутствие цен в блоке "Подобрали для вас"

**Приоритет:** P2 (Средний)  
**Серьёзность:** Minor

**Описание:**  
В блоке рекомендаций "Подобрали для вас" отсутствуют цены на объявления, хотя цена является важным фактором принятия решения.

**Фактический результат:**  
Отображаются только изображения без цен.

**Ожидаемый результат:**  
В каждой карточке рекомендации должна отображаться цена.

---

### BUG-009: P3 — Подозрительно низкая цена на породистого котёнка

**Приоритет:** P3 (Низкий)  
**Серьёзность:** Trivial

**Описание:**  
В пятой карточке "Экзотический короткошёрстный котёнок" указана цена 500 ₽, что подозрительно низко для породистого котёнка. Возможно, отсутствует валидация минимальной цены.

**Фактический результат:**  
Цена 500 ₽ для породистого котёнка.

**Ожидаемый результат:**  
Должна быть валидация минимальной цены для категории (например, минимум 1000 ₽).

---

### BUG-010: P3 — Некорректное отображение тултипа

**Приоритет:** P3 (Низкий)  
**Серьёзность:** Minor

**Описание:**  
Тултип "Не показывайте мне похожие объявления" отображается с неправильным позиционированием — накладывается на контент карточки.

**Фактический результат:**  
Тултип перекрывает цену и другие элементы карточки.

**Ожидаемый результат:**  
Тултип должен отображаться над или рядом с элементом, не перекрывая контент.

## Задание 2: Баги API

### BUG-API-001: Некорректная структура ответа при получении объявления

**Приоритет:** P1 (Высокий)  
**Серьёзность:** Major

**Описание:**  
Эндпоинт GET /api/1/item/{id} возвращает массив с одним элементом вместо объекта.

**Фактический результат:**
```json
[
  {
    "id": "...",
    "name": "...",
    ...
  }
]
```

**Ожидаемый результат:**
```json
{
  "id": "...",
  "name": "...",
  ...
}
```

**Обоснование:**  
REST API для получения одного ресурса по ID должен возвращать объект, а не массив. Текущая структура не соответствует REST-конвенциям.

---

### BUG-API-002: Различие в именовании полей sellerID vs sellerId

**Приоритет:** P2 (Средний)  
**Серьёзность:** Minor

**Описание:**  
В запросе на создание используется `sellerID`, а в ответе — `sellerId` (разный case).

**Фактический результат:**
- Запрос: `sellerID`
- Ответ: `sellerId`

**Ожидаемый результат:**  
Единообразное именование полей (рекомендуется camelCase: `sellerId`).

---

### BUG-API-003: Отсутствует код ответа 401 Unauthorized

**Приоритет:** P3 (Низкий)  
**Серьёзность:** Trivial

**Описание:**  
В документации отсутствует описание кода 401 Unauthorized для защищённых эндпоинтов (если такие планируются).

**Ожидаемый результат:**  
Документация должна содержать описание всех возможных кодов ответа.
